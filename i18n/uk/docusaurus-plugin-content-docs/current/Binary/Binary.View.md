# Binary.View


## Description

Створює або розширює двійковий файл із користувацькими обробниками для операцій запитів і дій.


## Syntax

```powerquery
Binary.View(
    binary as binary,
    handlers as record
) as binary
```


## Details

Повертає подання таблиці "<code>binary</code>", де функції, визначені параметром <code>handlers</code>, використовуються замість поведінки за замовчуванням, коли до цього подання застосовують певну операцію.<br />Якщо <code>binary</code> указано, усі функції обробника є необов’язковими. Якщо <code>binary</code> не вказано, функцію обробника <code>GetStream</code> необхідно вказати. Якщо для операції не вказано функцію обробника, до таблиці "<code>binary</code>" натомість застосовується поведінка за замовчуванням для цієї операції (окрім випадків застосування функції <code>GetExpression</code>).<br />Функції обробника мають повертати значення, семантично еквівалентне результату виконання операції для таблиці "<code>binary</code>" (або отримане подання, якщо використовується <code>GetExpression</code>).<br />Якщо функція обробника повертає помилку, до подання застосовується поведінка за замовчуванням для цієї операції.<br />Можна використати <code>Binary.View</code>, щоб запровадити зворотне перетворення до джерела даних – перетворити запити M на операції для конкретних джерел (наприклад, завантажити розділ файлу).<br />Перегляньте опубліковану документацію щодо настроюваного з’єднувача Power Query, щоб дізнатися більше про <code>Binary.View</code>.<br />


## Examples

### Example #1 
Створіть просте подання, яке не потребує доступу до даних для визначення довжини.
```powerquery
Binary.View(
    null,
    [
        GetLength = () => 12,
        GetStream = () => Text.ToBinary("hello world!")
    ]
)
```

Result: 
```powerquery
Text.ToBinary("hello world!")
```
