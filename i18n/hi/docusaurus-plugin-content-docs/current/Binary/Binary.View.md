# Binary.View


## Description

क्वेरी और कार्रवाइयों के निष्पादन के लिए उपयोगकर्ता-परिभाषित हैंडलर वाली बाइनरी बनाता है या विस्तारित करता है.


## Syntax

```powerquery
Binary.View(
    binary as binary,
    handlers as record
) as binary
```


## Details

<code>binary</code> का एक दृश्य लौटाता है, जिसमें दृश्य पर कार्रवाई के लागू होने पर, <code>handlers</code> में निर्दिष्ट फ़ंक्शन का उपयोग कार्रवाई के डिफ़ॉल्ट व्यवहार के स्थान पर किया जाता है.<br />यदि <code>binary</code> प्रदान किया जाता है, तो सभी हैंडलर फ़ंक्शन्स वैकल्पिक होते हैं. यदि <code>binary</code> प्रदान नहीं किया जाता है, तो <code>GetStream</code> हैंडलर फ़ंक्शन आवश्यक होता है. यदि किसी कार्रवाई के लिए हैंडलर फ़ंक्शन निर्दिष्ट नहीं किया जाता है, तो इसके बजाय <code>binary</code> पर कार्रवाई का डिफ़ॉल्ट व्यवहार लागू किया जाता है (<code>GetExpression</code>की स्थिति को छोड़कर).<br />हैंडलर फ़ंक्शन्स को ऐसा मान लौटाना चाहिए जो <code>binary</code> (या <code>GetExpression</code> की स्थिति में परिणामी दृश्य) के लिए कार्रवाई को लागू करने के परिणाम के सीमेंटिक रूप से समतुल्य हो.<br />यदि कोई हैंडलर फ़ंक्शन एक त्रुटि देता है, तो दृश्य पर कार्रवाई का डिफ़ॉल्ट व्यवहार लागू होता है.<br /><code>Binary.View</code> का उपयोग डेटास्रोत पर फ़ोल्डिंग कार्यान्वित करने – M क्वेरीज़ का स्रोत-विशिष्ट ऑपरेशंस में रूपांतरण करने (उदाहरण के लिए, किसी फ़ाइल का कोई अनुभाग डाउनलोड करने) के लिए किया जा सकता है.<br /> <code>Binary.View</code> के अधिक पूर्ण विवरण के लिए Power Query कस्टम कनेक्टर का प्रकाशित दस्तावेज़ देखें.<br />


## Examples

### Example #1 
कोई ऐसा मूल दृश्य बनाएँ जिसे लंबाई निर्धारित करने के लिए डेटा तक पहुँचने की आवश्यकता न हो.
```powerquery
Binary.View(
    null,
    [
        GetLength = () => 12,
        GetStream = () => Text.ToBinary("hello world!")
    ]
)
```

Result: 
```powerquery
Text.ToBinary("hello world!")
```
