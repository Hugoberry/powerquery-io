# Web.Contents


url से डाउनलोड की गई सामग्रियों को बाइनरी के रूप में लौटाता है.


## Syntax

```powerquery
Web.Contents(
    url as text,
    optional options as record
) as binary
```


## Remarks

बाइनरी के रूप में <code>url</code> से डाउनलोड की गई सामग्री वापस करता है. वैकल्पिक रिकॉर्ड पैरामीटर,<code>options</code>, अतिरिक्त गुणों को निर्दिष्ट करने के लिए प्रदान किया जा सकता है. रिकॉर्ड में निम्न फ़ील्ड हो सकती हैं:    <ul><li><code>Query</code> : एस्केपिंग के बारे में चिंता किए बिना, प्रोग्राम द्वारा URL में क्वेरी पैरामीटर जोड़ें.</li><li><code>ApiKeyName</code> : यदि लक्ष्य साइट में एक API कुंजी की धारणा है, तो इस पैरामीटर का उपयोग उस कुंजी पैरामीटर के नाम (न मान) को निर्दिष्ट करने के लिए किया जा सकता है जिसका उपयोग URL में किया जाना चाहिए. क्रेडेंशियल में वास्तविक कुंजी मान प्रदान किया गया है.</li><li><code>Headers</code> : इस मान को रिकॉर्ड के रूप में निर्दिष्ट करने से किसी HTTP अनुरोध पर अतिरिक्त शीर्ष लेख प्रदान होंगे.</li><li><code>Timeout</code> : इस मान को अवधि के रूप में निर्दिष्ट करना HTTP अनुरोध के लिए टाइमआउट को बदल देगा. डिफ़ॉल्ट मान १०० सेकंड है.</li><li><code>ExcludedFromCacheKey</code> : इस मान को सूची के रूप में निर्दिष्ट करने से इन HTTP शीर्षलेख कुंजियों को डेटा कैशिंग के लिए परिकलन का भाग होने से बाहर किया जा सकेगा.</li><li><code>IsRetry</code> : इस लॉजिकल मान को true के रूप में निर्दिष्ट करने से डेटा फ़ेच करते समय कैश में मौजूद किसी भी मौजूदा प्रतिसाद को अनदेखा कर दिया जाता है.</li><li><code>ManualStatusHandling</code> : इस मान को सूची के रूप में निर्दिष्ट करने से उन HTTP अनुरोधों के लिए कोई भी अंतर्निहित हैंडलिंग को रोका जा सकेगा जिनके प्रतिसाद में इनमें से एक स्थिति कोड है.</li><li><code>RelativePath</code> : इस मान को पाठ के रूप में निर्दिष्ट करने से अनुरोध करने से पहले उसे आधार URL में जोड़ता है.</li><li><code>Content</code> : इस मान को निर्दिष्ट करने से पोस्ट की सामग्री के रूप में विकल्प के मान का उपयोग करके पोस्ट पर प्राप्त होने वाले वेब अनुरोध को परिवर्तित करता है.</li></ul>    HTTP अनुरोध या तो GET (जब सामग्री निर्दिष्ट न हो) या POST (जब सामग्री हो) के रूप में किया जाता है. POST अनुरोध अनाम रूप से किए जा सकते हैं.    <br />    HTTP प्रतिसाद के शीर्षलेख बाइनरी परिणाम पर मेटाडेटा के रूप में उपलब्ध हैं. कस्टम डेटा कनेक्टर संदर्भ के बाहर केवल प्रतिसाद शीर्षलेख का सबसेट उपलब्ध है (सुरक्षा कारणों के लिए).    


## Examples

### Example #1 
RelativePath और Query विकल्पों का उपयोग करके &lt;code&gt;&#34;https://bing.com/search?q=Power+Query&#34;&lt;/code&gt; की सामग्री पुनर्प्राप्त करें. इन विकल्पों को डायनेमिक रूप से स्टैटिक बेस URL क्वेरी करने के लिए उपयोग किया जा सकता है.
```powerquery
let
    searchText = "Power Query"
in
    Web.Contents(
        "https://www.bing.com",
        [
            RelativePath = "search",
            Query = [q = searchText]
        ]
    )
```

Result: 
```powerquery
binary
```


### Example #2 
बाइनरी JSON पेलोड पास करके और JSON के रूप में प्रतिसाद को पार्स करते हुए URL के विरुद्ध पोस्ट निष्पादित करें.
```powerquery
let
    url = ...,
    headers = [#"Content-Type" = "application/json"],
    postData = Json.FromValue([x = 235.7, y = 41.53]),
    response = Web.Contents(
        url,
        [
            Headers = headers,
            Content = postData
        ]
    ),
    jsonResponse = Json.Document(response)
in
    jsonResponse
```

Result: 
```powerquery
table
```


### Example #3 
किसी ऐसे सुरक्षित URL से कनेक्ट करें जो किसी प्रमाणीकरण कुंजी को उसकी क्वेरी स्ट्रिंग के भाग के रूप में स्वीकार करता है. M में गुप्त कुंजी को हार्ड-कोडिंग करने के बजाय (जिससे सुरक्षा को खतरा हो सकता है)     कुंजी को M में अपना नाम (इसका मान नहीं) निर्दिष्ट करके, वेब API प्रमाणीकरण चुनकर और कुंजी मान को वेब API क्रेडेंशियल के भाग के रूप में दर्ज करके सुरक्षित रूप से प्रदान किया जा सकता है.    इस तरीके में उपयोग किए जाने पर, निम्न उदाहरण &lt;code&gt;&#34;https://contoso.com/api/customers/get?api_key=\*\*\*\*\*\*&#34;&lt;/code&gt; करने के लिए एक अनुरोध जनरेट करेगा.
```powerquery
Web.Contents("https://contoso.com/api/customers/get", [ApiKeyName="api_key"])
```

Result: 
```powerquery
binary
```
