# WebAction.Request


## Description

क्रियान्वित किए जाने पर, एक क्रिया निर्मित करता है, जो एक HTTP अनुरोध को बाइनरी मान के रूप में निष्पादित करने के परिणाम वापस करेगी.


## Syntax

```powerquery
WebAction.Request(
    method as text,
    url as text,
    optional options as record
) as action
```


## Details

ऐसी कार्रवाई बनाता है, जो निष्पादित होने पर HTTP का बाइनरी मान के रूप में उपयोग करके <code>url</code> के विरुद्ध <code>method</code> अनुरोध पूरा करने के परिणाम वापस करेगा.    वैकल्पिक रिकॉर्ड पैरामीटर, <code>options</code>, अतिरिक्त गुणों को निर्दिष्ट करने के लिए प्रदान किया जा सकता है. रिकॉर्ड में निम्न फ़ील्ड हो सकती हैं:    <ul><li><code>Query</code> : एस्केपिंग के बारे में चिंता किए बिना, प्रोग्राम द्वारा URL में क्वेरी पैरामीटर जोड़ें.</li><li><code>ApiKeyName</code> : यदि लक्ष्य साइट में एक API कुंजी की धारणा है, तो इस पैरामीटर का उपयोग उस कुंजी पैरामीटर के नाम (न मान) को निर्दिष्ट करने के लिए किया जा सकता है जिसका उपयोग URL में किया जाना चाहिए. क्रेडेंशियल में वास्तविक कुंजी मान प्रदान किया गया है.</li><li><code>Headers</code> : इस मान को रिकॉर्ड के रूप में निर्दिष्ट करने से किसी HTTP अनुरोध पर अतिरिक्त शीर्ष लेख प्रदान होंगे.</li><li><code>Timeout</code> : इस मान को अवधि के रूप में निर्दिष्ट करना HTTP अनुरोध के लिए टाइमआउट को बदल देगा. डिफ़ॉल्ट मान १०० सेकंड है.</li><li><code>ExcludedFromCacheKey</code> : इस मान को सूची के रूप में निर्दिष्ट करने से इन HTTP शीर्षलेख कुंजियों को डेटा कैशिंग के लिए परिकलन का भाग होने से बाहर किया जा सकेगा.</li><li><code>IsRetry</code> : इस लॉजिकल मान को true के रूप में निर्दिष्ट करने से डेटा फ़ेच करते समय कैश में मौजूद किसी भी मौजूदा प्रतिसाद को अनदेखा कर दिया जाता है.</li><li><code>ManualStatusHandling</code> : इस मान को सूची के रूप में निर्दिष्ट करने से उन HTTP अनुरोधों के लिए कोई भी अंतर्निहित हैंडलिंग को रोका जा सकेगा जिनके प्रतिसाद में इनमें से एक स्थिति कोड है.</li><li><code>RelativePath</code> : इस मान को पाठ के रूप में निर्दिष्ट करने से अनुरोध करने से पहले उसे आधार URL में जोड़ता है.</li><li><code>Content</code> : इस मान को निर्दिष्ट करने से उसकी सामग्रियाँ HTTP अनुरोध के मुख्य भाग के रूप में उत्पन्न हो जाएँगी.</li></ul>    <br />    ध्यान दें कि यह फ़ंक्शन अधिकांश संदर्भ में अक्षम है. इसके बजाय Web.Contents या Web.Headers का उपयोग करने पर विचार करें.    


## Examples

### Example #1 
Bing के विरुद्ध GET अनुरोध पूरा करें.
```powerquery
WebAction.Request(WebMethod.Get, "https://bing.com")
```

Result: 
```powerquery
Action
```
